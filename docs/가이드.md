## 2. GitHub Pages에 배포하기

### 2.1 저장소 생성

1. GitHub 에서 새 저장소 생성
2. 이름: `username.github.io` 형식
   - 예: `bandobando9535-glitch.github.io`
3. **Public** 으로 설정

### 2.2 Git 초기 설정

```bash
# 사용자 정보 설정
git config --global user.name "본인이름"
git config --global user.email "본인이메일@gmail.com"

# 저장소 초기화
git init
git add .
git commit -m "첫 커밋"

# GitHub 저장소 연결
git remote add origin https://github.com/username/username.github.io.git
```

### 2.3 브랜치 이름 확인

GitHub Pages 는 **`main`** 브랜치를 기본으로 사용합니다.

```bash
# master 를 main 으로 변경
git branch -m master main
```

### 2.4 GitHub Desktop 으로 Push

1. GitHub Desktop 설치 및 로그인
2. **File** → **Add local repository** → 프로젝트 폴더 선택
3. **Summary** 에 커밋 메시지 입력
4. **Commit to main** 클릭
5. **Publish repository** 또는 **Push origin** 클릭

### 2.5 GitHub Pages 설정

1. 저장소 **Settings** → **Pages** 이동
2. **Source**: `Deploy from a branch` 선택
3. **Branch**: `main` 선택, 폴더 `/ (root)`
4. **Save** 클릭
5. 1~2 분 후 `https://username.github.io` 에서 확인

## 3. Unity 게임 올리기

### 3.1 Unity WebGL 빌드

1. Unity 에서 **File** → **Build Settings**
2. Platform: **WebGL** 선택 → **Switch Platform**
3. **Player Settings** → **Publishing Settings**
4. **Compression Format**: **`Disabled`** (중요!)
   - `Brotli` 나 `Gzip` 을 사용하면 GitHub Pages 에서 오류 발생
5. Output Directory: `C:\works\portfolio\games`
6. **Build** 클릭

### 3.2 파일 구조

```
games/
├── Build/
│   ├── games.data
│   ├── games.framework.js
│   ├── games.loader.js
│   └── games.wasm
├── TemplateData/
│   ├── style.css
│   ├── favicon.ico
│   └── 기타 이미지 파일
└── index.html
```

### 3.3 포트폴리오에 게임 링크 연결

`index.html` 의 게임 섹션에서 링크 수정:

```html
<a href="games/index.html" class="btn-small" target="_blank">플레이하기</a>
```

### 3.4 GitHub 에 업로드

1. GitHub Desktop 에서 변경사항 확인
2. 커밋 → Push
3. `Ctrl + Shift + R` (강력 새로고침) 로 브라우저 캐시 삭제

## 4. 자주 발생하는 오류와 해결 방법

### 4.1 404 오류

**원인**: GitHub Pages 설정이 안 되어 있음

**해결**:
1. Settings → Pages 에서 `Deploy from a branch` 선택
2. Branch 를 `main` 으로 설정
3. Save 클릭

### 4.2 `Content-Encoding: br` 오류

**원인**: Unity 가 Brotli 압축으로 빌드했는데, GitHub Pages 가 이를 지원하지 않음

**해결**:
1. Unity Player Settings → Compression Format → **`Disabled`**
2. 다시 빌드
3. 파일 교체 후 Push

### 4.3 `Content-Encoding: gzip` 오류

**원인**: Gzip 압축 파일인데 서버가 압축 해제를 못 함

**해결**:
1. Unity Player Settings → Compression Format → **`Disabled`**
2. 다시 빌드
3. `games` 폴더 완전 삭제 후 새 파일 복사
4. Push

### 4.4 `Invalid or unexpected token` / `.gz 404`

**원인**: 이전 빌드 파일이 섞여 있음

**해결**:
1. `games` 폴더 안의 **모든 파일 삭제**
2. Unity 에서 Compression Format `Disabled` 로 빌드
3. 새 파일 복사
4. `index.html` 에서 `.gz`, `.br` 검색 → 없어야 함
5. Push 후 `Ctrl + Shift + R`

### 4.5 변경사항이 감지되지 않음

**원인**: `.gitignore` 설정 또는 Git 캐시 문제

**해결**:
```bash
git add -f games/
git status
```

## 5. 커스터마이징 팁

### 내용 수정

| 수정할 부분 | 파일 | 위치 |
|------------|------|------|
| 이름 | `index.html` | `<div class="logo">`, `<h1>` |
| 소개 | `index.html` | `#about` 섹션 |
| 기술 스택 | `index.html` | `.skills-grid` |
| 게임 | `index.html` | `.games-grid` |
| 연락처 | `index.html` | `#contact` 섹션 |
| 색상 | `style.css` | `#58a6ff`, `#27c93f` 등 |

### 새 게임 추가

1. Unity 에서 새 게임 빌드 → `games/게임이름/` 폴더에 저장
2. `index.html` 에 새 `.game-card` 추가
3. 링크: `href="games/게임이름/index.html"`
